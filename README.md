본 프로젝트는 콘솔 기반 오목(FiveEye) 및 오셀로(Othello) 게임을 C/C++로 구현한 학습용 포트폴리오 프로젝트입니다.  
자료구조 설계, 입력 처리, 보드 렌더링 등 기본기 중심의 로직 구현 능력을 보여주기 위해 제작되었습니다.


## 1. 프로젝트 개요

본 프로그램은 두 가지 보드 게임(오목, 오셀로)을 공통된 구조로 처리하도록 설계되었습니다.  
게임 규칙과 화면 출력 로직을 명확히 분리하여 유지보수성과 확장성을 확보하는 것을 목표로 하였습니다.

- 오목(FiveEye): 같은 돌 5개가 연속으로 연결되면 승리  
- 오셀로(Othello): 상대 돌을 양쪽에서 감싸면 해당 돌을 뒤집음  

두 게임 모두 동일한 방향 스캔 로직을 공유하며, 각 규칙에 맞게 처리 방식만 분기하여 수행합니다.


## 2. 프로그램 구조

프로그램은 크게 Logic과 Render 두 구조체로 구성되며,  
main 함수에서는 Logic과 Render 객체를 생성하고 Logic의 보드 데이터를 Render에 전달하는 형태로 동작합니다.

구성 요소  
- Logic: 게임 데이터 관리, 규칙 처리, 승리 판정, 오셀로 뒤집기 처리  
- Render: 화면 출력만 담당하는 렌더링 모듈  

이러한 모듈화 구조는 유지보수성과 확장성을 높여주며,  
향후 GUI 적용, AI 기능 추가, 보드 크기 변경 등 구조적 변화를 쉽게 반영할 수 있습니다.


## 3. Logic 구조 (게임 규칙 처리)

Logic 구조체는 다음과 같은 멤버로 구성됩니다.

- int turn: 현재 턴(흑/백)  
- int Rule: 게임 모드 (1 = 오목, 2 = 오셀로)  
- int Size: 보드 크기  
- int** dat: 동적 할당된 보드 배열  
- vector<pair<int,int>> mem: 방향 탐색 시 좌표를 임시 저장하는 버퍼  

핵심 기능  
1. 돌을 둘 수 있는지 판단 (오목 규칙·오셀로 규칙 통합 처리)  
2. 4개 방향(가로, 세로, 두 개의 대각선) 쌍을 탐색하여 mem에 좌표 저장  
3. 오목: 같은 색이 연속된 개수를 세어 승리 조건 판정  
4. 오셀로: mem에 쌓인 좌표를 기준으로 뒤집기 처리  
5. 돌 배치 후 턴 교체  


## 4. Render 구조 (화면 출력 처리)

Render 구조체는 Logic이 제공하는 dat 배열을 참조하여 화면 출력만 담당합니다.  
게임 규칙이나 데이터 구조를 알 필요 없이, 전달받은 정보만으로 보드를 출력합니다.

특징  
- Logic에서 dat, Rule, num만 넘기면 출력 가능  
- 로직과 출력이 완전히 독립되어 있어 구조가 단순하고 유지보수에 유리  
- GUI 환경(Window API, MFC 등)으로의 전환이 쉬움  


## 5. 개발 환경

- 언어: C/C++  
- 컴파일러: MSVC (Visual Studio)  
- 실행 환경: Windows Console (CMD, PowerShell, Terminal)  
- 지원 게임  
  - 오목: 16 x 16  
  - 오셀로: 8 x 8  


## 참고 사항

콘솔 출력은 PC의 폰트 설정에 따라 선과 돌의 정렬에 약간의 차이가 발생할 수 있습니다.  
이는 출력 형태의 차이일 뿐 프로그램 로직에는 영향을 주지 않습니다.
